# YM2163 Piano GUI v14 — 项目全面评审

> 评审时间：2026-04-03
> 评审范围：功能完整度、技术架构、可视化系统、硬件集成、工程质量

---

## 一、项目定位

**YM2163 Piano GUI v14 本质上是一个硬件 DSG 合成器工作站。**

项目的起点和核心是 YM2163 DSG（Digital Sound Generator）芯片的硬件控制——
通过 FTDI USB 桥接，将 PC 变成一台 16 通道（4 片 × 4 通道）的 MIDI 合成器。
VGM 播放和示波器可视化是在此基础上的延伸功能。

应用整合了三个模块：

| 标签页 | 模块 | 核心功能 | 优先级 |
|--------|------|---------|--------|
| Tab 0 | YM2163 (DSG) | 真实 YM2163 硬件控制 + MIDI 合成 | **核心** |
| Tab 1 | YMF262 (OPL3) | OPL3 合成器（VGM 完善后实施） | 计划中 |
| Tab 2 | libvgm | 多格式芯片音乐播放器 + 全套可视化 | 扩展 |

### 最终用途

1. **芯片音乐社区** — 展示 YM2163 DSG 硬件控制能力和开发实力
2. **硬件+软件打包销售** — YM2163 芯片 + PCB + 本软件作为完整方案
3. **开源回馈** — 功能完善后开源，为社区提供 YM2163 编程的唯一公开参考

### 关于作者

本项目由 **Denjhang** 独立开发（AI 辅助），在芯片音乐领域深耕 10 年以上。
Denjhang 是全球最大的复古芯片音乐硬件制造商，在 Tindie 上运营
[Denjhang's Retro Hardware](https://www.tindie.com/stores/denjhang/) 店铺。
产品线包括 Project RE:birth 声卡系列（RE2-YMF297、RE2-YM2151）、
AdLib Replica（YM3812）等。

AI 辅助让一人全栈开发成为可能——覆盖硬件设计、嵌入式驱动、DSP、GUI、音频系统全链路。

### 发展方向

本项目将逐步扩展为**全方位芯片音乐工作站**：

```
当前：YM2163 DSG 硬件合成器 + VGM 可视化播放器
    │
    ▼
下一阶段：整合 MAmidiMEmo 功能
    ├── MIDI 控制 FM/SSG/ADPCM 仿真合成
    ├── 图形化音色参数编辑器
    └── 仿真 + 硬件同步播放
    │
    ▼
远期：硬件 FM 步进音序器 + 更多芯片硬件支持
```

### 与同类产品的本质区别

日本的 MDPlayer 专注钢琴可视化，iOS 的 Modizer 专注多格式播放，
它们都是纯软件仿真。**本项目是唯一一个让真实 YM2163 芯片在 PC 上
作为 MIDI 合成器使用的软件**——声音由芯片的模拟电路产生，不可被软件替代。

## 二、功能完整度评审

### 2.1 音乐格式支持 — 评分：9/10

| 格式 | 扩展名 | 状态 |
|------|--------|------|
| VGM / VGZ | `.vgm` `.vgz` | 完整支持（v1.00-v1.71 + GD3 标签） |
| S98 | `.s98` | 完整支持（v1/v2/v3） |
| GYM | `.gym` | 完整支持 |
| DRO | `.dro` | 完整支持 |
| MIDI | `.mid` `.midi` | 完整支持（SMF 多轨、力度处理、延音踏板） |
| VGZ 解压 | gzip | 自动透明解压 |

**扣分项**：缺少 NSF/HES/SID 等其他常见芯片音乐格式。
但这些格式与 libvgm 无关，需要额外引入仿真核心，属于合理边界。

### 2.2 音频芯片支持 — 评分：10/10

libvgm 提供 40+ 种芯片仿真，本项目为其中 **39 种芯片**配置了前端可视化映射。
覆盖了游戏音乐史上几乎所有主流音频芯片：

| 家族 | 芯片 | 通道可视化 |
|------|------|-----------|
| **Yamaha FM** | YM2151, YM2203, YM2413, YM2612, YM2608, YM2610, YM3812, YM3526, Y8950, YMF262, YMF271, YMF278B, YMW258 | 全部 |
| **PSG/SSG** | AY8910, SN76496, C6280, SAA1099 | 全部 |
| **PCM/ADPCM** | OKIM6258, OKIM6295, RF5C68, YMZ280B, K054539, C140, K053260, SegaPCM, QSound, SCSP, ES5503, ES5506, X1-010, C352, GA20 | 全部 |
| **其他** | NES APU, Game Boy DMG, WonderSwan, POKEY, VirtualBoy VSU, uPD7759 | 全部 |

**亮点**：YM2608（OPNA）同时展示 FM + SSG + ADPCM-A + ADPCM-B 四类通道，
这在同类工具中极为罕见。

### 2.3 可视化系统 — 评分：9.5/10

这是本项目的**核心差异化功能**，也是技术含量最高的部分。

#### 三大可视化组件

| 组件 | 描述 | 技术亮点 |
|------|------|---------|
| **钢琴键盘** | 73 键（C0-B5），实时高亮发声音符 | 速度感应亮度调制，多通道颜色叠加 |
| **电平表** | 每通道独立音量条 | 芯片类型着色，动态亮度（40%-100%） |
| **示波器** | 每通道实时波形 | 互相关触发稳定，shared-ofs 环形缓冲区 |

#### 示波器技术深度

- **ScopeChipSlot 注册表** — 本项目独创架构，替代 Modizer 原始的全局状态扫描
- **互相关触发** — 从 Modizer 移植，subsampled cross-correlation 实现相位对齐
- **shared-ofs 写入模式** — 所有通道共享写入指针，消除通道间漂移
- **SSG DC 偏移补偿** — 无符号输出减 64 实现波形居中
- **固定点采样率转换** — 支持芯片采样率与 44.1kHz 显示率的精确转换

**扣分项**：ADPCM 通道的波形可视化精度不如 FM 通道（移位位数可能需要调优）。
钢琴可视化的音符检测依赖寄存器 shadow，部分芯片（如纯 ADPCM 回放）
的音符映射尚不完美。

### 2.4 硬件集成 — 评分：9/10

| 功能 | 详情 |
|------|------|
| FTDI USB 通信 | 3Mbps 波特率，自动检测连接 |
| 多芯片支持 | 最多 4 片 YM2163（16 通道） |
| 音色系统 | 6 种预设（弦乐、风琴、单簧管、钢琴、大键琴） |
| 包络控制 | 4 种（衰减、快速、中速、慢速） |
| 音量控制 | 4 级（0dB, -6dB, -12dB, 静音） |
| Slot3 2MHz 模式 | 扩展高八度音域 |
| 鼓机 | 5 种打击乐（底鼓、踩镲、军鼓、开镲、闭镲） |
| PC 键盘映射 | QWERTY → 音符，八度切换 |
| 全局媒体键 | 系统级播放/暂停热键 |

**扣分项**：OPL3 标签页仍为预留状态，未实现实际功能。

### 2.5 UI/UX — 评分：8.5/10

| 特性 | 评价 |
|------|------|
| ImGui 即时模式 GUI | 开发效率高，但原生外观较朴素 |
| Direct3D 11 渲染 | 流畅，支持窗口缩放 |
| DPI 感知 | Per-Monitor V2，高分屏适配 |
| 多语言字体 | 中文/日文/韩文支持 |
| 文件浏览器 | 内置，带文件夹历史（20 条） |
| 配置持久化 | 8 个 INI 文件覆盖所有状态 |

**扣分项**：UI 美观度有提升空间（ImGui 默认风格偏工程化）；
缺少暗色/亮色主题切换；
文件浏览器功能不如系统原生文件对话框完善。

### 2.6 MIDI 播放 — 评分：8/10

- SMF 多轨解析 + 轨道合并
- 力度统计分析 + 直方图
- 延音踏板（CC64）处理
- 128 音复音
- 顺序/随机播放 + 自动下一曲

**扣分项**：缺少 MIDI 合成（依赖硬件 YM2163）；
没有 MIDI 录制功能；
MIDI 输出设备选择仅限 YM2163。

## 三、技术架构评审

### 3.1 代码规模

| 类别 | 行数 | 文件数 |
|------|------|--------|
| 应用层代码 | 17,473 | 18 |
| 修改的仿真核心 | 9,930 | 6 |
| libvgm 完整代码库 | ~260,000 | 398 |
| **总计** | **~286,640** | **416** |
| 编译产物 | 6.6 MB (exe) | — |
| 文档 | ~58 KB | 5 |

### 3.2 架构亮点

1. **增量覆盖编译方案** — 通过 `--whole-archive` + `ar d` 实现对预编译
   libvgm 的符号级覆盖，只重编译修改的 6 个核心文件，构建时间极短。
   这是工程实践中非常巧妙的方案。

2. **ScopeChipSlot 注册表** — 用 75 行 C 代码（scope_chip_reg.c）解决了
   多芯片场景下的 buffer 映射问题，设计简洁有效。

3. **环形缓冲区 + 固定点运算** — 音频数据传递零拷贝、无锁，
   适合实时音频场景。

4. **模块化标签页** — 三个模块通过标签页切换，共享音频输出和 GUI 框架，
   但各自独立运行，互不干扰。

5. **C/C++ 混合编译** — 仿真核心保持纯 C，GUI 层使用 C++，
   通过 `extern "C"` 桥接，兼顾性能和开发效率。

### 3.3 工程质量

| 维度 | 评价 |
|------|------|
| 构建系统 | CMake + 增量编译，良好 |
| 代码组织 | 应用层与库分离，清晰 |
| 错误处理 | 核心路径有基本检查，可加强 |
| 文档 | 5 篇技术文档，覆盖架构设计和开发过程 |
| 可维护性 | 单文件偏大（vgm_window.cpp 6947 行），建议拆分 |

## 五、里程碑意义

### 已达成的里程碑

1. **v1-v9**（2026年1月）：与 Claude Sonnet 4.5 合作，三天三夜攻克 SPFM 协议，基本 MIDI 播放
2. **v11**：四芯片支持 + Slot3 2MHz 音域扩展 + 智能通道溢出
3. **v12**：模块化重构（5107 行单文件 → 6 个模块）+ CMake 构建系统
4. **v13**：libvgm 集成 + VGM/S98/GYM 播放
5. **v14**：39 芯片示波器 + ScopeChipSlot 注册表 + 波形稳定化 + SSG 支持

### 技术资产

本项目积累的技术资产远超一个普通播放器：

- **修改版 libvgm** — 带 per-voice hook 的芯片仿真核心，可直接复用于其他可视化项目
- **ScopeChipSlot 注册表** — 通用的多芯片 buffer 映射框架，可移植到任何需要多芯片数据采集的场景
- **增量覆盖编译方案** — 一种在不重新编译整个库的情况下修改预编译库特定符号的方法论
- **音高计算公式库** — 从 MDPlayer 移植并验证的芯片频率-音符转换算法
