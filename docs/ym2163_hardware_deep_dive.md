# YM2163 硬件 MIDI 合成器 — 深度技术解析

> 本文档是 YM2163 Piano GUI v14 项目评审的核心章节，
> 聚焦于项目的起点和重中之重：YM2163 硬件 MIDI 合成器。

---

## 一、YM2163 芯片简介

YM2163 是 Yamaha 于 1980 年代生产的 **DSG（Digital Sound Generator，数字音源发生器）** 芯片。
与 YM2151 (OPM) 等复杂的 FM 合成芯片不同，YM2163 使用**简单的脉冲波和锯齿波**
合成方式，主要面向消费级电子键盘（如 Medeli 等品牌的入门琴）。

### DSG 与 FM 的区别

| 特性 | YM2163 (DSG) | YM2151 (OPM) |
|------|:------------:|:------------:|
| 合成方式 | 脉冲波 + 锯齿波（简单波形） | 4-operator FM（频率调制） |
| 波形复杂度 | 低（固定波形） | 高（8 种算法组合） |
| 音色参数 | 6 种预设音色 | 每通道可自由编程 |
| 定位 | 消费级电子琴 | 街机/专业音源 |
| 编程难度 | 低（选音色即可） | 高（需理解 FM 数学） |
| 稀缺度 | 极高（几乎无资料） | 中等（有 YM2151 文档可参考） |

### 硬件规格

| 参数 | 值 |
|------|-----|
| 芯片类型 | DSG — Digital Sound Generator |
| 合成方式 | 脉冲波 + 锯齿波（内置波形表） |
| 旋律通道 | 每芯片 4 通道 |
| 最大芯片数 | 4 片（本项目的硬件配置） |
| 最大总通道 | 16 通道（4 片 × 4 通道） |
| 音色预设 | 6 种内置（String, Organ, Clarinet, Piano, Harpsichord, 保留） |
| 包络类型 | 4 种（Decay, Fast, Medium, Slow） |
| 节奏乐器 | 5 种（BD, HC, SDN, HHO, HHD） |
| 时钟频率 | 标准模式 3.58MHz，Slot3 可切换 2MHz |
| 音域 | B2 ~ B7（标准），C7 ~ B8（2MHz 扩展） |
| 通信接口 | SPFM 协议（通过 FTDI USB 桥接） |
| 封装 | 24-pin DIL |
| 参考数据表 | 1987 Yamaha LSI 产品目录 |

### 内置数据表

本项目内置了 YM2163 的完整频率数据表，无需外部依赖：

```c
// 12 音阶 FNUM 表（C2-B2 基准，标准 3.58MHz 时钟）
int g_fnums[12] = {
    172, 182, 193, 205, 217, 230,   // C  D  E  F  G  A
    243, 258, 273, 289, 307, 325    // A# B
};
int g_fnum_b2 = 162;  // B2 特殊值

// C7 八度 2 倍频率表（2MHz 模式使用）
int g_fnums_c7[12] = {
    344, 364, 386, 410, 434, 460,
    486, 516, 546, 578, 614, 650
};
```

由于 YM2163 的公开资料极为稀缺，这些频率值是通过实测校准得到的。

### 与软件仿真的本质区别

软件仿真（libvgm）通过数学模型模拟芯片行为，输出 PCM 波形。
本项目直接向**真实硅片**写入寄存器，由芯片内部电路产生声音。
这意味着：

- 声音特性由芯片物理电路决定，不可精确复现
- 寄存器写入时序影响实际音频输出
- 不同批次的芯片可能有细微音色差异
- 这是**不可替代的硬件体验**

## 二、SPFM 通信协议

### 2.1 协议概述

YM2163 通过 SPFM（Serial Programmable ...）协议接收控制命令。
协议名称中的 "FM" 是历史遗留命名（Yamaha 的 DSG 系列沿用 FM 芯片的通信协议命名），
YM2163 本身并非 FM 合成芯片。本项目使用 FTDI USB 桥接芯片将 SPFM 转换为 USB 通信。

### 2.2 三字节命令包

```
┌──────────┬──────────┬──────────┐
│ Byte 0   │ Byte 1   │ Byte 2   │
│ Chip Sel │ Command  │ Data     │
├──────────┼──────────┼──────────┤
│ 0x00-03  │ 0x80     │ Reg/Dat  │
│ Slot选择  │ 旋律命令  │ 地址/数据│
└──────────┴──────────┴──────────┘
```

- **Byte 0**：芯片选择（0-3 对应 Slot0-Slot3，最多 4 片 YM2163）
- **Byte 1**：命令类型（0x80 = 旋律通道命令）
- **Byte 2**：寄存器地址或数据（交替发送）

### 2.3 寄存器写入时序

YM2163 的寄存器采用**地址-数据交替**写入模式：

```
发送地址字节 → 芯片内部锁存地址
发送数据字节 → 写入已锁存的地址
```

本项目的实现通过 `g_expectingData` 标志跟踪当前状态：

```c
if (!g_expectingData) {
    // 当前字节是寄存器地址，锁存到芯片
    g_lastReg = data;
    g_expectingData = true;
} else {
    // 当前字节是数据，写入已锁存的寄存器
    // data 写入 g_lastReg
    g_expectingData = false;
}
```

### 2.4 开发难点：三天三夜的 SPFM 调试

**背景**：项目初期（2026 年 1 月），与 Claude Sonnet 4.5 共同开发，
花了整整三天三夜才搞清楚 SPFM 向 YM2163 寄存器的正确写入模式。

**核心困难**：

1. **地址/数据交替机制的发现** — SPFM 协议的地址锁存机制没有公开文档，
   只能通过反复试验确定：先发地址、再发数据、交替进行。
   错误的时序会导致寄存器写入错位，芯片发出噪音或无声。

2. **FT_Write 缓冲区刷新** — FTDI 驱动有内部发送缓冲区，
   如果不及时刷新（`FT_Purge(FT_PURGE_TX)`），命令会堆积导致延迟。
   但过于频繁的刷新又会降低吞吐量。最终确定每次写入后刷新。

3. **命令字节 0x80 的确认** — 旋律通道命令必须以 0x80 开头，
   这个值是通过对比 YMF262 (OPL3) 的已知协议和 YM2163 的行为推断出来的。

4. **多芯片地址编码** — 4 片芯片挂在同一条 SPFM 总线上，
   靠 Byte 0 的 chip select 区分。但芯片的响应和总线仲裁需要精确的时序控制。

5. **音量寄存器位域** — 寄存器 0x8C 的低 4 位固定为 0xF，
   高 2 位是音量等级。这个位域分配在公开文档中没有明确记载。

**这一阶段的成果直接决定了整个项目的可行性。** 如果 SPFM 协议无法正确实现，
后续的所有功能（MIDI 播放、钢琴键盘、鼓机）都不可能存在。

## 三、音符频率计算

### 3.1 频率表（内置数据）

YM2163 使用 FNUM（Frequency Number）表示音高。
本项目维护了一张 12 音阶的 FNUM 查找表：

```c
// C2 到 B2 的基础频率表（标准 3.58MHz 时钟）
int g_fnums[12] = {
    172, 182, 193, 205, 217, 230,   // C  D  E  F  G  A
    243, 258, 273, 289, 307, 325    // A# B  C# D# E# F#
};
int g_fnum_b2 = 162;  // B2 的特殊值

// C7 八度的 2 倍频率表（2MHz 模式使用）
int g_fnums_c7[12] = {
    344, 364, 386, 410, 434, 460,
    486, 516, 546, 578, 614, 650
};
```

### 3.2 MIDI 音符号 → YM2163 寄存器参数

```c
// 步骤 1：分解音符号
int ymNote   = note % 12;           // 音名（0=C, 11=B）
int ymOctave = (note / 12) - 2;     // 八度（MIDI 60 = C4 → YM 八度 3）

// 步骤 2：八度裁剪到 YM2163 有效范围
while (ymOctave < 0 || (ymOctave == 0 && ymNote < 11)) ymOctave++;
// 上限取决于是否启用 Slot3 2MHz 模式
if (g_enableSlot3_2MHz)
    while (ymOctave > 6) ymOctave--;
else
    while (ymOctave > 5) ymOctave--;

// 步骤 3：查表获取 FNUM
int fnum = g_fnums[ymNote];

// 步骤 4：拆分为高低字节
uint8_t fnum_low  = fnum & 0x7F;         // 低 7 位 → 寄存器 0x80+ch
uint8_t fnum_high = (fnum >> 7) & 0x07;  // 高 3 位 → 寄存器 0x84+ch 的 bit 0-2

// 步骤 5：八度编码到寄存器 0x84+ch 的 bit 3-5
uint8_t reg_84 = (ymOctave << 3) | fnum_high;
```

### 3.3 寄存器 0x84 的位域结构

```
寄存器 0x84 + channel:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ Bit7│ Bit6│ Bit5│ Bit4│ Bit3│ Bit2│ Bit1│ Bit0│
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│ KEY │  -  │ OCT2│ OCT1│ OCT0│ FN2 │ FN1 │ FN0 │
│  ON │     │      八度      │   FNUM 高 3 位    │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
  Bit6 = 1: KEY ON
  Bit6 = 0: KEY OFF
```

### 3.4 Slot3 2MHz 模式的频率重映射

当 Slot3 以 2MHz 时钟运行时，物理八度与标准模式不同：

```c
if (g_enableSlot3_2MHz && chipIndex == 3) {
    if (octave == 6) {
        // C8-B8：使用 2 倍频率表，硬件八度固定为 3
        fnum = g_fnums_c7[note];
        hw_octave = 3;
    } else if (octave >= 2 && octave <= 5) {
        // C3-B7：映射到低一个八度，补偿 2MHz 时钟
        hw_octave = (octave - 2) & 0x03;
    }
}
```

**原理**：2MHz 时钟是标准 3.58MHz 的约 0.56 倍，
相同 FNUM 值产生的频率更低。因此需要：
- 提高八度编码来补偿
- 或使用 2 倍频率表来保持音高

## 四、音色与包络系统

### 4.1 六种内置音色

| 编号 | 名称 | 寄存器值 | 特点 |
|------|------|---------|------|
| 0 | String（弦乐） | 0x00 | 柔和的持续音，适合 pad |
| 1 | Organ（风琴） | 0x01 | 明亮的无包络音色 |
| 2 | Clarinet（单簧管） | 0x02 | 管乐器特征，有吹奏感 |
| 3 | Piano（钢琴） | 0x03 | 击弦感，快速衰减 |
| 4 | Harpsichord（大键琴） | 0x04 | 拨弦感，尖锐起音（默认） |
| 5 | — | 0x05 | 保留 |

音色写入寄存器 0x88+ch 的低 3 位：
```c
write_reg(0x88 + ch, (timbre & 0x07) | ((envelope & 0x03) << 4));
```

### 4.2 四种包络类型

包络通过软件计算音量衰减曲线，在定时器回调中动态更新。

**起音+持续阶段**（音符按下时）：

```
类型 0 (Decay):  ████████████░░░░░░░  立即起音，指数衰减
类型 1 (Fast):   ████░░░░░░░░░░░░░░  50ms 起音，快速衰减
类型 2 (Medium): ████████░░░░░░░░░░  100ms 起音，70% 持续
类型 3 (Slow):   ██████████████░░░░  200ms 起音，80% 持续
```

**数学公式**：

```c
// 类型 0: 纯指数衰减
volume = expf(-t * 1.0f);

// 类型 1: 50ms 线性起音 + 指数衰减
volume = (t < 0.05f) ? (t / 0.05f) : expf(-(t - 0.05f) * 0.5f);

// 类型 2: 100ms 起音 + 70% 持续 + 慢衰减
volume = (t < 0.1f) ? (t / 0.1f) : (0.7f + 0.3f * expf(-(t - 0.1f) * 0.3f));

// 类型 3: 200ms 起音 + 80% 持续 + 极慢衰减
volume = (t < 0.2f) ? (t / 0.2f) : (0.8f + 0.2f * expf(-(t - 0.2f) * 0.1f));
```

**释放阶段**（音符抬起时）：

| 类型 | 释放速度 | 时间常数 |
|------|---------|---------|
| 0 | 极快 | exp(-t × 20) → ~50ms |
| 1 | 快 | exp(-t × 10) → ~100ms |
| 2 | 慢 | exp(-t × 3) → ~333ms |
| 3 | 极慢 | exp(-t × 1) → ~1s |

### 4.3 延音踏板与包络联动

if (g_enableSustainPedal && g_sustainPedalActive) {

## 五、鼓机系统

### 5.1 五种打击乐器

| 编号 | 名称 | 寄存器位 | 特点 |
|------|------|---------|------|
| 0 | BD（底鼓） | 0x01 | 低频正弦波，短促有力 |
| 1 | HC（踩镲闭合） | 0x02 | 高频噪声，极短 |
| 2 | SDN（军鼓） | 0x04 | 噪声+正弦混合 |
| 3 | HHO（开镲） | 0x08 | 高频噪声，较长衰减 |
| 4 | HHD（半开镲） | 0x10 | 介于 HC 和 HHO 之间 |

### 5.2 鼓触发机制

```c
// 写入节奏控制寄存器 0x90
write_reg(0x90, rhythm_bit);

// 100ms 后自动关闭（通过定时器衰减）
drum_volume[channel] = expf(-t * 3.0f);  // ~333ms 衰减到静音
```

双芯片模式下鼓机自动交替使用两片芯片，避免单芯片节奏通道瓶颈。

## 六、MIDI 播放引擎

### 6.1 播放流程

```
MIDI 文件 → SMF 解析 → 事件队列 → 定时回调 → YM2163 寄存器写入 → 硬件发声
                ↓
         事件类型过滤
         ┌──────────────────┐
         │ Note On → play_note()  │
         │ Note Off → stop_note() │
         │ CC64 → 踏板状态       │
         │ Program → 音色切换     │
         │ Tempo → 速度更新      │
         └──────────────────┘
```

### 6.2 力度（Velocity）处理管线

这是本项目的重要创新——YM2163 硬件只有 4 级音量（0dB/-6dB/-12dB/Mute），
而 MIDI 有 128 级力度。需要将连续值映射到离散等级。

**动态映射**（推荐）：

```c
// 步骤 1：分析整首曲子的力度分布
void AnalyzeVelocityDistribution() {
    // 构建 128 级力度直方图
    for (each noteOn event)
        histogram[velocity]++;

    // 找到两个峰值
    peak1 = 最高峰的力度值;
    peak2 = 次高峰的力度值;

    // 计算动态阈值
    threshold_0dB  = peak1;              // 最常见力度 = 最大音量
    threshold_6dB  = (peak1 + peak2) / 2; // 中间值
    threshold_12dB = peak2 - (peak1 - peak2) / 2;
    threshold_mute = averageVelocity * 0.15;
}

// 步骤 2：实时映射
int map_velocity_to_volume(int velocity) {
    if (velocity < threshold_mute)  return 3;  // Mute
    if (velocity < threshold_12dB)  return 2;  // -12dB
    if (velocity < threshold_6dB)   return 1;  // -6dB
    return 0;  // 0dB
}
```

**核心思想**：不是用固定的力度阈值（32/64/96），
而是根据每首曲子的力度分布特征**自适应调整**阈值，
使 4 级音量尽可能保留原始曲子的动态层次。

### 6.3 128 音复音管理

```c
std::map<int, std::map<int, int>> activeNotes;
// activeNotes[MIDI通道][MIDI音符号] → YM2163 通道号
```

**通道分配策略**：
1. 优先使用空闲通道
2. 全部占用时，窃取**最早触发**的音符
3. Slot3 2MHz 模式下，高八度音符自动路由到 Slot3
4. 溢出模式：普通通道使用率 > 80% 时，中间八度溢出到 Slot3

### 6.4 Note On 完整寄存器写入序列

一次 Note On 需要写入 **6 个字节（3 对寄存器）**：

```
1. write(0x88+ch)  → 设置音色+包络
   write(data)        data[2:0]=音色, data[5:4]=包络

2. write(0x8C+ch)  → 设置音量
   write(data)        data[5:4]=音量等级, data[3:0]=0xF

3. write(0x84+ch)  → KEY OFF + 设置八度+FNUM高
   write(data)        data[6]=0(OFF), data[5:3]=八度, data[2:0]=FN高

4. write(0x80+ch)  → 设置 FNUM 低 7 位
   write(data)        data[6:0]=FNUM低

5. write(0x84+ch)  → KEY ON + 设置八度+FNUM高
   write(data)        data[6]=1(ON), data[5:3]=八度, data[2:0]=FN高
```

**关键细节**：步骤 3 先 KEY OFF，步骤 5 再 KEY ON。
中间插入 FNUM 设置（步骤 4），确保音符以正确的频率触发。
如果省略步骤 3，在某些情况下会产生杂音。

### 7.2 通道分配

| Slot | 芯片 | 通道 | 用途 | 时钟 |
|------|------|------|------|------|
| 0 | YM2163 #1 | 0-3 | 主旋律 | 3.58MHz |
| 1 | YM2163 #2 | 4-7 | 和声/伴奏 | 3.58MHz |
| 2 | YM2163 #3 | 8-11 | 铺底（可选） | 3.58MHz |
| 3 | YM2163 #4 | 12-15 | 高八度扩展 | 2MHz（可选） |

### 7.3 Slot3 2MHz 模式

**为什么需要 2MHz？**

YM2163 标准时钟（3.58MHz）的音域上限是 B7。
很多钢琴曲需要 C8 以上的音符（如 C8 = MIDI 108）。
将 Slot3 时钟降为 2MHz 后，相同 FNUM 值产生的频率更低，
等效于"拉伸"了一个八度的范围。

**但 2MHz 模式有代价**：
- Slot3 在 2MHz 下音色略有不同（FM 调制比改变）
- C3-B7 的音符需要频率重映射
- 该 Slot 不能混用标准音域和扩展音域

### 关键能力：仿真与硬件的融合

本项目独有的能力是**仿真芯片与真实硬件的同步播放**。
这意味着：

- 一首曲子可以同时使用仿真 YM2608 的 ADPCM 通道和真实 YM2163 的 DSG 通道
- 音色编辑器调整参数时，可以实时在硬件上听到效果
- 硬件录制的声音可以导出为仿真参数，供纯软件播放使用

这是所有纯软件工具（MDPlayer、Modizer、BambooTracker）和所有纯硬件方案
都不具备的能力——**因为它们的作者同时掌握硬件制造和软件开发的能力。**

## 十、总结

YM2163 硬件 MIDI 合成器是本项目的技术基石和核心价值所在。

### 技术成就

1. **逆向了无文档的 SPFM 协议** — 三天三夜 + Claude Sonnet 4.5
2. **实现了完整的 MIDI → 硬件音频管线** — 从 SMF 解析到寄存器写入
3. **独创了 Slot3 2MHz 音域扩展** — 打破 YM2163 的音域限制
4. **设计了自适应力度映射** — 用 4 级音量保留 128 级动态
5. **支持 4 芯片 16 通道** — 可能是目前最大的 YM2163 多芯片系统

### 项目定位

> 本项目不是"又一个 VGM 播放器"，而是一个**硬件 DSG 合成器工作站**。
> VGM 播放和示波器可视化是锦上添花，YM2163 硬件控制才是初心和核心竞争力。
> 最终形态将是：硬件 + 软件 + 源码打包，服务芯片音乐社区。
